üíª Embedding Instructions

To embed the widget on your website, add this script tag to your HTML:

<script src="http://localhost:8000/widget/video-banner-widget.js" data-api-url="http://localhost:8000"> </script>
Note: Replace http://localhost:8000 with your actual backend URL in production.

‚öôÔ∏è Backend Setup

Before the widget works, you need to:

  1. Start the backend server (if not already running):
    cd backend ./run.sh
  2. Create a store document in MongoDB with your domain:
    db.stores.insertOne({ domain: "localhost", name: "Test Store", widgetConfig: { videoUrl: "https://commondatastorage.googleapis.com/gtv-videos-bucket/sample/BigBuckBunny.mp4", clickableLink: "https://example.com" } })
Tip: For testing, you can use the domain "localhost" or your actual domain name. The widget will automatically use window.location.hostname to detect the domain.

üìä Analytics Events

The widget tracks three events:

  • widget_loaded: Fired when the widget script initializes
  • video_loaded: Fired when the video element is ready to play
  • banner_clicked: Fired when the user clicks the video banner

All events are sent to: POST /api/widget/analytics

Check your browser's Network tab to see the analytics requests.

üîç Troubleshooting

  • Widget not appearing? Check the browser console for errors
  • Video not loading? Verify the video URL is accessible and the domain is registered in MongoDB
  • Analytics not working? Check that the backend server is running and MongoDB is connected
  • CORS errors? Ensure the backend CORS settings allow your domain
